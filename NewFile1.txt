1. 사전 기획의 관점에서 프로젝트 완성도 평가
8.5
기획 단계에서 의도했던 핵심 기능들 (사용자 맞춤 여행일정 추천, 무작위 여행 일정 추천, 여행 일정 저장 및 여행 일정 추천, 무작위 여행 일정 추천, 여행 일정 저장 및 조회 기능, 여행 후기 작성 및 조회 기능)을 모두 구현함.
사용자 선택 옵션에 따라 채팅창 배경의 변화는 사용자의 여행에 대한 몰입도를 높이고, 사이트의 전체적인 컨셉과 목적성에 부합했다.
몇가지 아쉬운 점은 디자인의 디테일과 일관성, 예외처리의 미흡함이 있다. 실시간 관광 데이터의 연동 부족은 추후 확장 가능성으로 남는다.

2. 프로젝트 중 잘한 점과 아쉬운 점
잘한점
1. 구조적인 설계 및 컴포넌트 아키텍처
 1) useContext와 custom Hook 기반의 글로벌 상태 관리를 구현하여 앱 전반에서 데이터를 일관되게 관리할 수 있도록 했다.
 2) 이를 통해 복잡한 props 전달 없이 효율적인 상태 흐름을 가능하도록 했다.
 3) lazy-loading 및 라우팅 구조를 적용하여 페이지 전환 시 초기 렌더링 비용을 줄이고 사용자 흐름을 고려한 구조를 구현했다.
 4) 반복되는 로직은 컴포넌트로 분리하고, 조건부 렌더링을 적용하여 성능을 최적화 했다.
 
 2. OpenAI API 연동과 Prompt Engineering 적용
 1) GPT-3.5 API를 연동하여 사용자 입력값에 맞춘 일정 자동 생성기능을 구현했다.
 2) Prompt Engineering을 통해 여행 일정을을 일관된 형식으로 응답받도록 프롬프트를 세밀하게 설계하여 일반적인 
    채팅 느낌이 아닌 도매인의 특화된 형식으로 구현했다.

3. 백엔드 서버 및 보안 구조
 1) Node.js,  Express, MySQL을 기반으로 서버를 구성했다.
 2) 서버와 프론트의 연동을 위해 RESTful API를 설계했다.
 3) JWT(Json Web Token) 기반 인증 로직과 커스텀 미들웨어를 적용하여, 토큰 발급(1시간 유효시간), 비로그인 시 기능 제한(플랜 저장 및 마이페이지, 여행 후기 작성), 토큰 만료 시 로그아웃 처리 등 기본적인 보안 체계를 구축했다.

4. Git 기반 협업 및 문제 해결 
1) 협업 툴을 사용하여 개발을 분업하고 코트를 병합함으로써 프로젝트 코드의 전체적인 품질과 개발 흐름을 일관성 있게 유지하였다.
2) 코드 충돌 상황에서 커밋 히스토리 분석과 stash 등의 협업 도구를 적극적으로 활용하여 문제상황 해결했다.

5.여행 컨셉에 맞춘 감성적 UI/UX 디자인
  1) 챗봇 메시지 비행기 애니메이션, 여행 감성 문구 등 감성적 디자인을 추가하였다.
  2) Scroll 이벤트 등으로 부드러운 등장 효과 구현했다.

아쉬운 점
1) 실제 여행지 정보(운영 시간, 축제 일정, 기상 정보 등)가 반영되지 않아 현실성과 정확도에서 미흡함
2) GPT 응답만으로 일정을 생성하다 보니 실제 지역 정보, 날씨, 운영 시간 등의 실데이터 연동은 미흡함
3) 챗봇 의존성 과다사용자의 재요청이나 세부 수정 요청을 효율적으로 반영하기엔 챗봇 방식만으로 한계가 있었음
4) 사용자 경험 관련 디테일(로딩 처리, 예외 메시지, 접근성 등)이 미완성된 부분이 있었음.
5) 향후 직접 수정 가능한 일정 편집 UI 필요

3. 향후 개선 및 보완 방향
1) 실데이터 연동: 한국관광공사 API, Google Maps, 지역 축제 일정 등 실제 데이터를 기반으로 추천하도록 로직 변경.
2)UI/UX 강화
3) 사용자가 직접 입력을 통해 저장 된 일정을 수정 및 저장할 수 있는 기능을 추가하도록 한다.
4) 사용자의 여행 스타일, 과거 플랜 등을 학습하여 더 정교한 추천이 가능한 맞춤형 시스템을 구현한다.

4 프로젝트 수행 과정에서의 성과와 느낀 점
이번 프로젝트를 통해 단순 구현이 아닌 기획부터 개발, 테스트와 오류 해결까지 프로젝트를 전반적으로 참여하며 경험했다. 
복잡한 기능을 통합하기도 하고, OpenAI API 연동, 라우터 기반 모달 설계, 서버 구현과 DB설계까지 작은 기능들이지만 실제 서비스에 가까운 구조 설계 경험을 할 수 있었다.
전체적인 프로세스를 참여함으로 풀스택 개발자로서의 성장 가능성과 자신감을 얻었다.

결론
이번 프로젝트는 스스로에게 학습으로서도 의미가 크지만 그것을 넘어 실제 서비스로 발전 가능한 잠재력을 갖추고 있다고 생각된다.
여러가지 아쉬운 부분이 남지만 개선하고 기능 확장을 통해 많은 방향으로 발전 할 수 있다고 생각한다.
다양한 기술을 사용하면서 학습하고 기획부터 구현까지 직접 경험한 소중한 시간이었다.
협업을 통해 여러상황에서의 커뮤니케이션 스킬에 대한 중요성을 배우고 실무에서 요구되는 팀 프로젝트 운영 방식의 이해도도 향상되었다.
이 경험을 바탕으로 향후 실무에서도 문제 해결과 사용자 중심 설계에 강점을 가진 개발자로 성장해 나가고 싶다.
